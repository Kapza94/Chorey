# Chorey Design Context

## Visual Direction

Use the supplied design handoff as the baseline, originally named Sprout. Rename visible product language to Chorey.

Chorey should feel warm, clean, tactile, and focused. Product UI should serve repeated household tasks: creating chores, checking progress, approving work, and reviewing balances.

## Color

- Use creamy off-white surfaces rather than pure white.
- Use warm near-black text rather than pure black.
- Keep the three money buckets visually consistent:
  - Spend: warm peach
  - Savings: soft lilac
  - Giving: sage green
- Color carries meaning, not decoration.
- Avoid rainbow palettes, glass effects, and chrome gradients.

## Typography

- Use readable sans typography for app UI.
- Money should use tabular numbers and always show two decimal places.
- Keep labels direct and sentence case.
- Avoid display-style typography inside dense app controls.

## Components

- Parent UI should be calm and task-focused.
- Child UI can be slightly larger and warmer.
- Use familiar controls for forms, toggles, tabs, and settings.
- Use Lucide-style outlined icons or native equivalents consistently.
- No emoji as product UI icons.

## Motion

- Use short, purposeful transitions.
- Motion should confirm state changes, especially chore completion and approval.
- No confetti or noisy celebration in MVP.

## Layout

- Mobile-first for iOS and Android.
- Bottom tabs for main parent and child surfaces.
- Bucket totals should be prominent on child dashboard.
- Chore rows should stay simple and scannable.
